# 竞品分析工作流

## 1. 触发条件

当用户出现以下意图时，进入本流程：

- 做竞品分析
- 拆解参考产品
- 输出差异分析
- 想先看竞品，再反推产品设计

## 2. 默认读取文件

- `docs/ui-system/竞品分析&产品专家（用于竞品分析及 PRD方案输出）.md`
- `docs/prompts/competitor-analysis-prompt-template.md`
- `docs/ui-system/ui-interaction-spec.md`
- `docs/ui-system/erp-reference-patterns.md`

## 3. 输入信息

优先收集以下内容：

- 竞品名称或链接
- 分析目标
- 分析维度
- 输出用途
- 是否要继续转产品方案

## 4. 执行步骤

1. 明确竞品对象与分析范围
2. 对齐分析目标和输出深度
3. 输出竞品分析
4. 提炼差异点与机会点
5. 询问是否继续转产品方案、PRD 或原型

## 5. 默认产物

- 竞品分析文档
- 差异总结
- 机会点建议

当前优先采用用户提供的“竞品分析&产品专家”主提示词来驱动上述产物。

## 6. 结束动作

分析输出后，必须追加一句明确确认：

“竞品分析已完成，是否继续基于分析结论转产品方案、PRD 或可交互原型？”
