# 产品经理工作流总控

## 1. 文档定位

本文件定义当前项目的总控路由逻辑，用于在新对话中快速判断用户意图，并分发到对应工作流。

本文件不是执行提示词，而是路由说明。

## 2. 四类主路径

### 2.1 新需求方案与 PRD

适用场景：

- 用户有新需求，想先梳理背景与方案
- 用户希望输出 PRD
- 用户还不确定是否需要立即生成原型

默认结果：

- 需求对齐结论
- 方案建议
- PRD 文档
- 询问是否继续生成原型

当前主提示词来源：

- `docs/ui-system/跨境电商IT产品专家（用于新需求方案输出）.md`

### 2.2 竞品分析

适用场景：

- 用户想拆解竞品
- 用户想看竞品差异与机会点
- 用户想在竞品分析后继续做产品方案

默认结果：

- 竞品分析文档
- 差异总结
- 机会点与策略建议
- 询问是否继续转 PRD / 原型

当前主提示词来源：

- `docs/ui-system/竞品分析&产品专家（用于竞品分析及 PRD方案输出）.md`

### 2.3 基于 PRD 生成原型

适用场景：

- 用户已经有 PRD
- 用户希望直接生成页面原型
- 用户希望输出 HTML 可交互原型

默认结果：

- 页面结构拆解
- 原型说明
- HTML 可交互原型

### 2.4 日常讨论 / 评审 / 修改

适用场景：

- 用户只想交流
- 用户想评审某份文档
- 用户想修改现有方案

默认结果：

- 讨论结论
- 修改建议
- 如用户要求，可升级为正式产物

## 3. 路由优先级

### 3.1 意图明确时

如果用户已经明确说了要做哪类事情，不再重复提问，直接进入对应工作流。

### 3.2 意图不明确时

优先使用以下开场问题：

“你这次主要想做哪类事情：新需求方案与 PRD、竞品分析、基于 PRD 生成原型，还是日常讨论？”

## 4. 状态推进规则

### 4.1 新需求方案流

状态推进：

1. 对齐背景
2. 对齐问题与目标
3. 输出方案
4. 输出 PRD
5. 询问是否继续生成原型

### 4.2 竞品分析流

状态推进：

1. 明确竞品对象
2. 明确分析维度
3. 输出分析
4. 提炼机会点
5. 询问是否继续转 PRD / 原型

### 4.3 PRD 转原型流

状态推进：

1. 获取 PRD
2. 读取规范
3. 拆页面与交互
4. 生成 HTML 原型

### 4.4 日常沟通流

状态推进：

1. 回答问题
2. 评审或修改
3. 仅在用户明确要求时，转入结构化产出

## 5. 当前实现说明

当前项目的“Router Agent”效果由以下三部分共同实现：

- `AGENTS.md`
- 本文件
- 各工作流文件与提示模板

如果后续你希望进一步做成更像“真 Skill”的形式，可以再把这些流程抽成正式技能包，但当前这版已经能稳定支撑项目级工作流。
