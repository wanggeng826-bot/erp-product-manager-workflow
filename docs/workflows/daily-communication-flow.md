# 日常讨论 / 评审 / 修改 工作流

## 1. 触发条件

当用户出现以下意图时，进入本流程：

- 日常沟通交流
- 评审已有方案
- 修改现有文档
- 讨论产品问题

## 2. 默认读取文件

按问题需要读取，不强制读取全部规范。

如果讨论内容涉及现有项目规范、PRD 或原型，再补充读取对应文件。

## 3. 执行原则

- 以快速解决问题为主
- 不强制用户进入结构化文档产出流程
- 仅当用户明确要求时，再升级成 PRD、竞品分析或原型任务

## 4. 默认产物

- 讨论结论
- 评审意见
- 修改建议

## 5. 升级条件

若用户在交流中明确提出以下需求，则切换流程：

- 要求输出 PRD：切到“新需求方案与 PRD 工作流”
- 要求做竞品分析：切到“竞品分析工作流”
- 要求基于 PRD 生成原型：切到“PRD 转原型工作流”
