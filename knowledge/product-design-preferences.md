# Product Design Preferences

## UI 风格

- 默认使用中文 B 端 ERP 后台风格。
- 使用 Ant Design 作为组件和交互基线。
- 优先左侧导航、顶部栏、面包屑、紧凑页面标题、浅灰背景、白色内容面板。
- 信息密度保持中高，适合长期高频操作。
- 表格、筛选、状态标签、抽屉、表单、任务列表优先于装饰性卡片和大图表。
- 卡片圆角克制，通常不超过 8px。
- 图表用于辅助判断，不作为页面视觉主角。

## 避免方向

- 避免欧美 SaaS 风格。
- 避免营销看板式首屏。
- 避免过大留白、过大标题、过强渐变、强装饰背景。
- 避免正式原型中出现演示控制、角色切换器、页面状态切换器、调试面板。
- 避免新增 PRD 未要求、用户未确认的页面模块。
- 避免正式原型中出现无法映射回 PRD 的导航项、页签、摘要卡、快捷操作和页面说明文案。
- 避免把“状态覆盖”和“权限差异”做成页面里的测试开关或演示面板。

## PRD 输出偏好

- 研发可落地。
- 新需求必须先做背景对齐、信息缺口、确认清单和输出策略确认。
- 用户确认前，不直接输出方案草案、PRD 草案、原型方案或创建业务文档。
- 需要明确业务背景、目标用户、核心场景、范围边界、功能说明、权限、状态、异常、验收标准。
- 复杂 PRD 默认分段高保真输出。
- 如果信息不确定，先进入确认清单，再继续推进。

## 原型输出偏好

- HTML 可交互原型优先。
- 原型必须能映射回 PRD 或用户确认需求。
- 页面结构优先体现真实业务操作，而不是展示效果。
- 查看详情优先抽屉，少跳转。
- 列表页优先支持筛选、表格、批量操作、状态反馈、详情入口。
- 若页面清单未冻结，先做来源映射，再生成原型。

## 报表设计偏好

- 报表涉及时间对比时，默认提供按天、按周、按月三个时间维度。
- 时间对比默认包含标准对比模式和自定义对比模式。
- 标准对比模式按当前统计周期自动计算上一周期环比和同比周期。
- 按日选择完整月份时，上月同比应对应完整上月。
- 报表指标卡片需要在鼠标悬浮时展示本期、环比、同比或自定义对比的时间范围和值。
- 店铺控件沿用当前 ERP 的 BI 驾驶舱样式，层级固定为平台、事业部、站点、店铺名。
- 店铺控件平台层必须展示平台图标和平台名称。

## UI 优化偏好

- 先判断是否符合中文 B 端 ERP 期待。
- 先找结构问题，再看视觉 polish。
- 输出问题时按 `P0 / P1 / P2` 区分严重程度。
- 优化建议需要可执行，不只写“更清晰”“更美观”。
- 中间方案沟通阶段默认不做 Playwright 二次审核；只在 PRD 或原型接近终稿、或用户明确要求最终 QA 时再启用自动化审核。
