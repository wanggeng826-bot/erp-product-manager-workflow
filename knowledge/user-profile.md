# User Profile

## 已确认

- 用户希望把 Codex 打造成产品经理工作流助手，而不是一次性问答工具。
- 用户偏好先搭好规则、Skill、知识库，再让后续 PRD、竞品分析、原型生成持续复用。
- 用户关注输出是否符合自己的产品口味，不希望每次生成结果都偏离既定标准。
- 用户希望新需求、竞品分析、PRD、原型、UI 优化能形成可调用的工作流。
- 用户希望原文类提示词和规范尽量保持不动，项目只做索引和路由。
- 用户重视长对话下的上下文稳定，希望通过文件化计划和知识库降低遗忘风险。

## 输出偏好

- 需要先判断任务类型，再进入对应流程。
- 需要明确区分短期进度记录和长期知识沉淀。
- 对方案输出要求偏研发可落地，不喜欢泛泛而谈。
- 对 UI 原型要求偏中文 B 端 ERP 后台，不喜欢欧美 SaaS 或营销看板风格。
- 对代码和原型文件修改要求克制、可验证、最小改动。

## 待确认

- 用户在公司内的正式岗位和决策权限。
- 用户希望知识库是否记录个人管理风格、沟通风格和评审偏好。
- 用户是否允许沉淀公司内部更细的组织、权限、业务流程信息。
