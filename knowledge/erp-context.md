# ERP Context

## 已确认

- 产品类型：自研 ERP。
- 使用端：B 端后台。
- 目标用户：公司全员。
- 高频动作：查询、编辑、操作。
- UI 基线：Ant Design。
- 页面体验要求：尽量少跳转。

## 交互约束

- 查看详情优先使用抽屉。
- 查阅类抽屉默认占屏幕宽度约 2/3。
- 新建字段少时使用弹窗。
- 新建字段多、联动多、步骤多时使用抽屉或独立页面。
- 删除必须二次确认。
- 重要操作必须可撤回，或明确说明不可撤回原因。
- 不同角色看到的按钮和可操作范围不同。
- 权限差异应通过真实 UI 状态体现，例如隐藏、禁用、只读和反馈。

## 操作日志约束

操作日志只能基于既定模板做删减，不能新增字段。

允许字段：

- 菜单
- 一级功能
- 二级功能
- 操作人
- 操作类型
- 操作对象
- 操作详情
- 操作时间

## 待确认

- ERP 当前已有模块清单。
- 角色权限矩阵。
- 审批、导入导出、任务中心、通知中心等公共能力是否已有标准实现。
