# 主体管理企查查同步与工商变更保护 PRD

> 当前版本：V1.0  
> 日期：2026-05-20  
> 原型位置：`prototype/principal-management/index.html`  
> 在线原型：https://wanggeng826-bot.github.io/erp-product-manager-workflow/prototype/principal-management/  
> 适用系统：基础配置系统 / 主体管理  

## 1. 背景

现有主体管理已支持主体基础信息的新增、编辑、删除、启停、关联店铺查看和操作日志。当前主体数据主要依赖 ERP 内部人工维护，存在工商信息滞后、漏改、与外部工商权威信息不一致的问题。

本次需求引入企查查企业工商详情接口，将企查查作为主体工商信息的权威来源，支持大陆企业、香港企业的工商信息同步，并预留海外企业支持能力。个人主体不调用企查查。

同时需要处理一个关键业务场景：账号管理部可能已向工商局提交主体变更资料，并在 ERP 内先行维护了目标信息，但企查查仍返回变更前数据。此时不能简单用企查查旧数据覆盖 ERP 内部正在办理中的目标值，因此需要引入“工商变更办理中保护”机制。

## 2. 目标

1. 建立主体与企查查企业数据的匹配关系，保存企查查唯一识别码 `QccCode`。
2. 对大陆企业、香港企业进行企查查工商信息同步。
3. 默认以企查查返回数据自动覆盖 ERP 工商字段，减少人工维护滞后。
4. 支持“工商变更办理中”保护，用户可录入保护截止日期，并可再次编辑。
5. 超过保护截止日期后进行提醒，支持继续延长、撤回或标记失败。
6. 完整记录同步日志、字段覆盖日志、跳过覆盖原因、保护编辑日志。

## 3. 范围

### 3.1 本期范围

- 主体列表增加企查查匹配状态、工商同步状态、保护截止日期、最后同步时间。
- 主体类型支持：大陆企业、香港企业、海外企业、个人。
- 大陆企业、香港企业支持企查查匹配和同步。
- 个人主体标记为不适用，不调用企查查。
- 海外企业先保留“待确认支持”状态，待接口覆盖范围确认后再启用同步。
- 主体详情增加工商信息、工商变更记录、同步日志、关联店铺。
- 支持工商变更办理中保护设置。
- 支持保护超期提醒和再次编辑。

### 3.2 非本期范围

- 不处理个人主体的外部实名或证照核验。
- 不接入其他国家工商数据库。
- 不自动发起工商局变更办理流程。
- 不替代账号管理部线下审核和材料流转。

## 4. 用户角色

| 角色 | 权限 |
| --- | --- |
| 账号管理部 | 维护主体信息、提交工商变更保护、编辑保护截止日期、处理超期 |
| 财务/运营 | 查看主体工商信息、查看关联店铺、查看同步结果 |
| 系统管理员 | 配置同步策略、手动触发同步、查看全部日志 |
| 系统任务 | 定时调用企查查接口并执行自动覆盖或跳过覆盖 |

## 5. 核心规则

### 5.1 主体类型与调用规则

| 主体类型 | 是否调用企查查 | 说明 |
| --- | --- | --- |
| 大陆企业 | 是 | 使用统一社会信用代码、企业名称或 `QccCode` 查询 |
| 香港企业 | 是 | 使用企业名称、企业编号、商业登记号码或 `QccCode` 查询 |
| 海外企业 | 待确认 | 先保留状态，不默认调用 |
| 个人 | 否 | 标记为不适用，不调用企查查 |

### 5.2 字段命名

原“统一社会信用代码”字段调整为“注册/信用代码”。

- 大陆企业：展示统一社会信用代码。
- 香港企业：展示商业登记号码，必要时展示企业编号。
- 海外企业：后续按接口支持范围定义。
- 个人：可为空或不适用。

### 5.3 权威数据规则

企查查数据是工商信息权威来源。默认情况下，企查查同步成功后，系统自动覆盖 ERP 中的工商字段。

ERP 内部字段分为：

| 类型 | 字段示例 | 是否被企查查覆盖 |
| --- | --- | --- |
| 工商权威字段 | 企业名称、法定代表人、注册/信用代码、登记状态、注册地址、经营范围、注册资本、成立日期、登记机关、核准日期、企业类型、`QccCode` | 是 |
| ERP 管理字段 | 主体启停状态、备注、关联店铺、创建人、更新人、保护状态、保护截止日期 | 否 |

### 5.4 工商变更办理中保护

当用户在 ERP 内手动修改工商敏感字段，并选择“工商变更办理中”原因时，系统创建保护记录。

保护记录必须填写：

- 变更原因
- 工商提交日期
- 预计完成日期 / 保护截止日期
- 保护字段
- 办理状态
- 备注，选填
- 证明材料，选填

保护期内：

1. 企查查同步仍正常调用。
2. 系统保存企查查原始快照。
3. 非保护字段正常自动覆盖。
4. 命中保护字段时不覆盖 ERP 当前值。
5. 日志记录“因工商变更保护跳过覆盖”。

### 5.5 自动解除保护

当企查查返回值与 ERP 当前目标值一致时：

1. 对应字段自动标记为已完成。
2. 所有保护字段均一致后，保护记录自动变为已完成。
3. 主体工商同步状态恢复为正常。
4. 记录自动解除保护日志。

### 5.6 保护超期

当当前日期超过保护截止日期，且企查查返回值仍未与 ERP 当前目标值一致：

1. 主体工商同步状态变为“保护已超期”。
2. 列表、详情、同步日志中展示超期提醒。
3. 默认继续保护字段，不自动覆盖。
4. 用户可再次编辑保护记录，延长截止日期或更新办理状态。

超期处理动作：

- 延长保护截止日期。
- 标记工商变更审核失败。
- 标记已撤回。
- 撤回 ERP 修改并按企查查数据覆盖。

## 7. 企查查接口

接口文档：https://openapi.qcc.com/dataApi/735

使用要点：

- `searchKey` 支持企业名称、统一社会信用代码、注册号或 `QccCode`。
- `QccCode` 是全球各类商业实体唯一识别码。
- 香港企业返回 `No` 作为企业编号。
- 香港企业返回 `CreditCode` 作为商业登记号码。
- `ChangeRecords` 返回工商变更记录。

## 9. 验收标准

1. 个人主体不会触发企查查调用。
2. 大陆企业、香港企业可匹配企查查并保存 `QccCode`。
3. 企查查同步成功后，非保护工商字段自动覆盖 ERP 字段。
4. 存在工商变更保护时，保护字段不被企查查旧值覆盖。
5. 用户可录入和再次编辑保护截止日期。
6. 超过保护截止日期后，列表展示“保护已超期”。
7. 保护超期后默认继续阻止保护字段覆盖。
8. 企查查返回值与 ERP 目标值一致时，系统自动解除保护。
9. 每次同步、覆盖、跳过覆盖、保护编辑均有日志。
10. 关联店铺列表保持现有能力，不受企查查同步影响。
