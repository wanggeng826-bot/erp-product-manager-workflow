# Shopee 授权流程改造方案

> 版本：v2.0 | 日期：2026-05-19 | 状态：待评审
> 基于：《店铺重构方案》+ Shopee Open Platform 开发者文档 + BigSeller 授权实践
> 
> **前提确认：** ERP 已在 Shopee Open Platform 注册 Developer App，partner_id / partner_key 已获取，回调域名已配置，Token 刷新模块已就绪。

---

## 1. 改造背景

### 1.1 现状问题

Shopee 店铺授权与 Amazon / Lazada 等平台存在根本性差异，不能简单套用"填 API Key → 验证 → 完成"的统一授权模型：

- Shopee 采用 **OAuth 2.0 跳转授权**，用户必须离开 ERP 到 Shopee 页面完成登录和店铺选择；
- Shopee 区分 **店铺账号（单店）** 和 **主账号（多店批量）** 两种授权路径；
- 主账号授权后返回的是 `main_account_id`，ERP 需要通过 API 拉取旗下的 `shop_id_list`；
- **Shopee 授权返回的是平台原始店铺数据，不是 ERP 内的子店铺记录**——用户需要在 ERP 侧逐个或批量"认领/录入"这些店铺成为可运营的子店铺。

### 1.2 改造目标

| 目标 | 说明 |
|---|---|
| 授权合规 | 严格对齐 Shopee Open Platform OAuth 2.0 规范 |
| 子店铺录入 | 授权完成后，提供清晰的子店铺认领与录入流程 |
| 主账号/店铺账号双路径 | 支持两种授权方式，并自动识别返回数据差异 |
| 用户不困惑 | 每个步骤明确告知当前在做什么、下一步做什么 |

## 2. 核心概念澄清

### 2.1 Shopee 账号层级

```
Shopee 卖家中心
├── 主账号（Main Account）
│   ├── 子店铺 A（MY 站点）
│   ├── 子店铺 B（SG 站点）
│   ├── 子店铺 C（TH 站点）
│   └── 商户（Merchant）—— 跨境卖家才有
│
└── 店铺账号（Shop Account）—— 独立店铺，无主账号
```

### 2.2 ERP 侧对应关系

| Shopee 概念 | ERP 侧概念 | 说明 |
|---|---|---|
| 主账号 | 授权凭证（不直接对应店铺） | 一次授权带回多个店铺，需用户逐一认领 |
| 子店铺 | **店铺（Store）** | ERP 中 4 级层级下的叶子节点 |
| 商户（Merchant） | 跨境店铺的附加凭证 | 用于调用 Merchant API（全球商品、库存推送等） |
| 站点 | **站点（Site）** | 店铺层级中平台→事业部→站点→店铺名的第 3 级 |

### 2.3 "授权"≠"创建店铺"

**授权**是获取 Shopee 平台的数据访问权限（拿到 token + shop_id 列表）。

**录入子店铺**是用户将授权返回的 shop_id 匹配到 ERP 店铺层级（事业部、站点、店铺别名、运营/客服负责人）。

> ⚠️ 这是本次改造最核心的交互设计点：授权成功后，不能自动批量创建店铺，而要让用户**有选择地认领**需要纳入 ERP 管理的子店铺。

## 3. 完整流程：新增 Shopee 店铺 + 授权 + 子店铺录入

### 3.1 流程总览

```
┌──────────┐    ┌──────────┐    ┌──────────────┐    ┌──────────────┐
│ Step 1   │    │ Step 2   │    │   Step 3     │    │   Step 4     │
│ 填写基础  │───>│ 跳转Shopee│───>│  授权成功回调  │───>│  子店铺录入   │
│ 店铺信息  │    │ 完成授权  │    │  解析授权结果  │    │  认领+补全   │
└──────────┘    └──────────┘    └──────────────┘    └──────────────┘
   ERP 侧         Shopee 侧        ERP 后端处理        ERP 侧交互 ★
```

### 3.2 Step 1：填写店铺基础信息

用户在 ERP 店铺管理页点击「新增店铺」，选择平台为 **Shopee**：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 平台 | 只读 | — | 已选 Shopee |
| 店铺昵称（别名） | 输入框 | 是 | ERP 内部识别名，后续可修改 |
| 店铺类型 | 下拉 | 是 | `本土店` / `跨境店(CNSC)` / `3PF` |
| 事业部 | 下拉 | 是 | 默认预选当前用户所属事业部 |
| 运营负责人 | 下拉 | 否 | 可后续在子店铺录入时指定 |
| 客服负责人 | 下拉 | 否 | 可后续指定 |

填写完成后，点击 **「下一步：前往 Shopee 授权」**。

**交互要点：**

- 「前往 Shopee 授权」按钮点击后调用后端接口实时生成签名授权 URL，`window.open()` 新窗口打开；
- 授权链接签名有效期仅 5 分钟，如果用户在引导页停留过久需提示重新生成；
- 「授权完成，下一步」按钮在用户授权成功后由以下机制触发：
  - **方案一（推荐）**：回调页通过 `postMessage` 通知 ERP 主窗口授权成功，自动跳转至 Step 4；
  - **方案二**：用户手动点击「授权完成，下一步」，ERP 后端轮询校验是否已收到回调。

### 3.4 Step 3：回调处理（后端自动，用户无感）

Shopee 授权完成后，页面跳转至 ERP 配置的 redirect_uri，携带参数：

- **店铺账号授权**：`?code=xxx&shop_id=xxx&state=xxx`
- **主账号授权**：`?code=xxx&main_account_id=xxx&state=xxx`

ERP 回调页（一个轻量中间页，如 `/shopee/callback`）执行以下逻辑：

```
1. 解析 URL 参数，提取 code + shop_id（或 main_account_id）
2. 调用 POST /api/v2/auth/token/get 换取初始 token
3. 如果是主账号授权：
   a. 从响应中获取 shop_id_list 和 merchant_id_list
   b. 对每个 shop_id 调用 GET /api/v2/shop/get_shop_info 获取基础信息（店铺名称、站点、状态）
4. 将授权结果暂存至 session/临时表
5. 通过 postMessage 或重定向通知前端跳转至"子店铺录入"页面
```

**回调中间页展示（过渡态）：**

```
┌──────────────────────────────────────────┐
│                                          │
│         ✓ 授权成功！正在处理...            │
│                                          │
│   已获取店铺数据，即将进入店铺录入...        │
│                                          │
└──────────────────────────────────────────┘
```

#### 3.5.2 子店铺认领列表的三种状态

| 状态 | 条件 | 展示方式 | 用户可操作 |
|---|---|---|---|
| **已存在关联** | Shopee shop_id 已绑定到 ERP 已有店铺 | 绿色标识"已存在关联店铺"，展示已绑定的 ERP 店铺名 | 取消关联（仅限本次），或确认覆盖绑定 |

#### 3.5.5 提交校验规则

| 校验项 | 规则 | 错误提示 |
|---|---|---|
| 别名唯一性 | 所有认领店铺的别名不可重复，也不可与已有店铺别名重复 | "店铺别名 'xxx' 已被使用，请修改" |
| 别名非空 | 所有已勾选的新店铺必须填写别名 | "请填写店铺别名" |
| 事业部必填 | 所有已勾选的新店铺必须选择事业部 | "请选择事业部" |
| 至少认领一个 | 如果所有店铺都取消勾选，不允许提交 | "请至少认领一个店铺" |

#### 3.5.6 提交后处理

```
用户点击「确认录入所选店铺」
  → 前端校验通过
  → 调用后端接口，传入所有已勾选店铺的录入信息
  → 后端处理：
      1. 为每个新店铺创建 Store 记录（绑定 site_id、platform_shop_id）
      2. 创建对应的 shopee_token 记录（关联授权信息）
      3. 对已存在关联的店铺，更新绑定关系（如果需要）
      4. 记录操作日志
  → 返回结果，跳转至店铺列表
  → 列表刷新，新增的店铺出现在列表中，授权状态 = "已授权"
```

#### 3.5.7 "稍后录入"场景

如果用户选择了「跳过，稍后录入」：

- 授权结果（token + shop_id_list）已保存，但未创建店铺记录；
- 这些待认领的 shop_id 进入 **「待认领店铺池」**；
- 用户后续可以在以下位置重新进入认领：
  - 店铺列表顶部横幅提示：「您有 3 个 Shopee 店铺尚未录入，[前往认领]」
  - 店铺管理页 → 「待认领店铺」入口（或授权中心 → 待处理授权）

## 4. 主账号 vs 店铺账号：子店铺录入差异

### 4.1 店铺账号授权后

│  请确认以下店铺信息并纳入 ERP 管理：                      │
│                                                      │
│  店铺名称：My Shop (SG)                               │
│  站点：SG                                            │
│  平台店铺ID：12345                                    │
│                                                      │
│  店铺别名：[                    ]                     │

主账号授权可能返回多个 `shop_id`（甚至跨站点），使用 3.5 节描述的完整认领列表。

## 5. 已有店铺的"补充授权"场景

### 5.1 场景描述

- 用户已通过某主账号授权，录入了 3 个子店铺；
- 后来该主账号下新增了 2 个子店铺（如扩展了新站点），需要补充授权。

### 5.2 流程

```
店铺列表 → 找到已授权的主账号关联店铺 → 操作列「补充授权」
→ 跳转 Shopee 授权页（同一主账号再次授权）
→ 回调后 ERP 对比新旧 shop_id_list
→ 自动识别新增的 shop_id
→ 进入"子店铺录入"页，仅展示新增的 2 个店铺
→ 用户认领录入
```

**关键点**：补充授权时，已录入的店铺不重复出现在认领列表中（自动排除）。

## 9. 数据库设计补充

### 9.1 店铺表新增 Shopee 字段

```sql
-- store 表新增字段
ALTER TABLE store ADD COLUMN platform_shop_id VARCHAR(32);     -- Shopee shop_id
ALTER TABLE store ADD COLUMN platform_merchant_id VARCHAR(32); -- Shopee merchant_id
ALTER TABLE store ADD COLUMN main_account_id VARCHAR(32);      -- 关联的 Shopee 主账号
ALTER TABLE store ADD COLUMN is_sip TINYINT DEFAULT 0;         -- 是否 SIP 店铺
ALTER TABLE store ADD COLUMN auth_type VARCHAR(20);            -- 'shop_account' | 'main_account'
```

### 9.2 授权 Token 表

```sql
CREATE TABLE shopee_token (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  store_id BIGINT,                    -- 关联 ERP 店铺 ID（可为空，待认领状态）
  platform_shop_id VARCHAR(32),      -- Shopee shop_id
  merchant_id VARCHAR(32),           -- Shopee merchant_id
  main_account_id VARCHAR(32),       -- 主账号 ID
  auth_type VARCHAR(20),            -- 'shop_account' | 'main_account'
  access_token TEXT,                 -- 加密存储
  refresh_token TEXT,               -- 加密存储
  token_expire_at DATETIME,          -- access_token 过期时间
  refresh_expire_at DATETIME,        -- refresh_token 过期时间
  auth_expire_at DATETIME,           -- 授权有效期截止时间
  claim_status VARCHAR(20) DEFAULT 'pending',  -- 'pending' | 'claimed' | 'abandoned'
  auth_time DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 10. 验收标准

### 10.1 授权 + 子店铺录入全流程

- [ ] 新增 Shopee 店铺 → 填写基础信息 → 跳转 Shopee 授权 → 回调成功 → 进入子店铺录入页
- [ ] 主账号授权：展示多个子店铺，用户可逐个勾选认领，填写别名/事业部/负责人后提交
- [ ] 店铺账号授权：展示单店铺确认页，用户可确认录入或跳过
- [ ] 批量操作（全选/批量设置事业部/负责人）正常生效
- [ ] 勾选店铺提交后，ERP 自动创建对应的店铺记录，列表可查
- [ ] 取消勾选的店铺进入待认领池，后续可重新认领
- [ ] 别名重复校验生效
- [ ] 授权链接过期（5 分钟），提示用户重新生成

### 10.2 已有店铺重授权 / 补充授权

- [ ] 已过期店铺点击「重新授权」，完成 Shopee 授权后状态恢复
- [ ] 主账号关联店铺点击「补充授权」，新增子店铺出现在认领列表，已录入的不重复出现

### 10.3 待认领店铺管理

- [ ] 待认领店铺池正确展示所有未录入的 shop_id
- [ ] 可从待认领池发起单个或批量认领
- [ ] 授权过期后，待认领店铺标记为不可认领
