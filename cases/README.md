# Cases

本目录只存放具体业务案例。案例文件不是主线工作流的一部分，也不是新任务的默认输入。

## 使用规则

- 新需求默认不读取本目录。
- 只有用户明确说明“继续某个案例”“参考历史案例”“基于某个版本迭代”时，才读取对应子目录。
- 每个案例必须独立成目录：`cases/<case-name>/`。
- 案例内的 PRD、原型、评审材料、截图和版本记录不得写回主线 `intake/prd/`、`prototype/`、`knowledge/` 或 `skills/`。

## 开源仓库说明

本开源版本不包含真实业务历史案例。

如需新增案例，请使用脱敏后的示例，并放入：

```text
cases/<case-name>/
```
