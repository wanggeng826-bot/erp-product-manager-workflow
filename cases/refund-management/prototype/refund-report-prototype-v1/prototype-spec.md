# 原型说明

## 页面信息

- 页面名称：退款报表主页面（总览看板 / 明细报表 / 导出任务中心）
- 页面编号：REFUND-REPORT-L1
- 原型等级：L3
- 对应 PRD：`../../intake/prd/自研ERP退款报表_PRD_最终版_v2.docx`

## 1. 页面目标

在同一业务入口中承载退款总览看板、退款明细报表、导出任务中心与详情抽屉，满足以下目标：

- 统一退款率、成功率、完成态、退款时效与金额口径
- 支持管理层、运营、财务、客服等角色在少跳转前提下完成分析、下钻、查单、导出与核对
- 体现店铺权限、字段权限、按钮权限与导出权限差异
- 补齐默认态、加载态、空状态、错误态、无权限态与成功反馈态

## 2. 用户入口

- ERP 菜单：`报表中心 > 退款报表`
- 首次进入默认打开“退款总览看板”页签
- 看板中的指标卡、趋势图、平台/站点、SKU、原因、责任归因均支持下钻至“退款明细报表”
- 明细中的订单号 / 退款单号点击后打开右侧详情抽屉

## 3. 页面结构

- 页面标题区：标题、实时刷新标识、最近同步时间、口径与权限提示
- 筛选与控制区：角色切换、页面状态切换、共享筛选模型、高级筛选展开
- 页签内容区：
  - `退款总览看板`
  - `退款明细报表`
  - `导出任务中心`（按权限显示）
- 详情抽屉：基础信息、金额与状态、原因与审计三个分组页签
- 辅助弹层：
  - 列管理弹窗
  - 金额修正确认弹窗
  - 导出任务抽屉
  - Notification / Message 风格提示

## 4. 核心操作

- 查询 / 重置：总览与明细共用查询快照
- 高级筛选展开：补充业务类型、处理类型、金额类型、原因、责任归因、发货仓
- 看板下钻：点击图点、平台、站点、SKU、原因、责任方后自动切到明细并带参
- 明细查看：点击平台订单号或退款单号打开抽屉，不跳新页
- 列管理：按当前角色权限裁剪可见字段，默认关键列不可全部取消
- 异步导出：冻结当前筛选快照创建任务，完成后在任务中心 / 抽屉查看结果
- 高风险逆向动作：在财务/管理层/管理员角色下支持“金额修正”，二次确认后可立即撤销

## 5. 权限差异

- 可见角色：
  - 看板、明细：销售、运营、客服、财务、管理层、开发/问题治理、管理员
  - 导出任务中心：运营、财务、管理层、管理员
- 可操作角色：
  - 查询、下钻、详情抽屉：全业务角色
  - 异步导出：运营、财务、管理层、管理员
  - 金额修正：财务、管理层、管理员
- 只读角色或字段：
  - 销售 / 客服 / 开发默认隐藏财务核账字段
  - 销售 / 客服 / 开发默认不开放导出
  - 客服导出策略原型按“默认关闭 / 待确认”展示禁用态
  - 财务核账字段对财务、管理层、管理员为只读可见

## 6. 状态变化

- 默认态：展示最近 30 天、全部授权店铺、总览看板
- 加载态：Skeleton 模拟看板与明细加载
- 空状态：保留当前筛选，提供“重置筛选”入口
- 错误态：展示错误说明与重试入口
- 成功反馈态：查询刷新、导出创建、导出完成、复制、撤销均有轻提示
- 禁用态：
  - 客服导出按钮可见但禁用，并说明“策略待确认”
  - 无需感知该能力的角色直接隐藏入口，例如销售 / 开发的导出与任务中心
- 无权限态：展示 Result 风格说明，区分菜单权限与数据权限

## 7. 页面跳转

- 无独立新页面跳转，采用同页签 + 抽屉闭环
- 看板下钻：
  - 指标卡 -> 明细页签
  - 趋势图日期点 -> 明细页签并透传日期点
  - 平台/站点/组合 -> 明细页签并透传对应维度
  - SKU / 原因 / 责任方 -> 明细页签并透传筛选条件
- 明细详情：
  - 订单号 / 退款单号 -> 详情抽屉
- 导出结果：
  - 明细操作区 -> 导出任务抽屉 / 导出任务中心

## 8. 备注

- 操作日志展示严格遵循模板字段：
  - 菜单
  - 一级功能
  - 二级功能
  - 操作人
  - 操作类型
  - 操作对象
  - 操作详情
  - 操作时间
- 原型使用本地模拟数据与前端逻辑演示口径、权限与交互闭环，不代表真实接口结构
- 推荐评审顺序：
  1. 切换角色看权限差异
  2. 切换页面状态看完整性
  3. 从看板任一区块执行下钻
  4. 在明细页执行列管理、详情查看、异步导出
  5. 在财务角色下验证金额修正与撤销
