# 销售人员管理原型上下文压缩

## 当前目标

基于《销售人员管理 V1.2》需求生成并持续优化 HTML 原型，路径：

- `/Users/freddy/Documents/New project/prototype/salesperson-management/index.html`

## 已确认设计规则

- 左侧树是业务上下文切换，不默认跨模块跳转。
- 在销售目标模块中切换业务单元时，应加载对应业务单元年度目标设置。
- 顶部全局区只放面包屑和模块上下文，不把业务单元名称作为所有模块的全局标题。
- 模块级导航放在内容区顶部左侧，不放右上角。
- 具体操作按钮必须放回对应功能区：
  - 添加/编辑人员：成员列表工具区
  - 新建调整任务：历史订单调整工具区
  - 保存目标配置：销售目标工具区
- 行内编辑必须显式点击“编辑”，再暂存/取消，不允许单击展示值直接编辑。
- 抽屉内局部交互不能整抽屉重建导致滚动跳动。
- 截图作为需求来源时，必须同时提取知识和样式，不只提取文案。
- 销售目标设置遵循“业务单元总盘子 -> 人员承接 -> 店铺可选拆解”。人员目标之和不能超过业务单元目标；需要拆到店铺的人员，店铺目标必须平齐人员目标。
- 销售目标树表格必须支持行内展开/收起，BU 展开人员、人员展开店铺；差额集中在汇总列，不在月度单元格到处铺红条。

## 年度目标导入方案

- 三阶段：下载模板、上传文件、结果反馈。
- 四原则：效率、准确性、易用性、反馈。
- 下载模板阶段应展示模板版本、模板预览、使用提示、文件约束、Excel 校验方法、错误报告方案。
- 上传阶段应展示新增/覆盖模式、上传组件、文件解析状态、三层校验：
  - 模板结构校验
  - 单行数据格式校验
  - 业务逻辑校验
- 上传后创建后台任务，用户去任务中心追踪。
- 错误报告默认方案二：单 Sheet + 状态列，在原始数据右侧追加“是否导入成功”和“错误原因”。

## 已沉淀到 Skill/规范的规则

- 文件导入统一三阶段与任务中心结果下载。
- 截图需求必须映射知识、布局、视觉层级、颜色语义和交互状态。
- 模块级导航不放右上角。
- 左侧树切换默认保持当前模块，仅刷新范围数据。
- 功能按钮必须贴近所属功能区，避免脱离上下文。
