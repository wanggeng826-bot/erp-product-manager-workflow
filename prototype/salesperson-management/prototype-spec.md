## 3. 交互覆盖

- 成员列表可勾选人员，批量按钮根据选择状态启用/禁用。
- 店铺列点击后按平台分组展示店铺明细。
- 添加/编辑人员弹窗内店铺资源池支持树形勾选、半选态和冲突归属识别。
- 保存人员配置时，如包含他人负责店铺，触发归属转移二次确认。
- 转岗弹窗提供“仅人员转岗”和“携店铺转岗”两个明确业务按钮。
- 移除人员时根据名下店铺数量展示强警告。
- 操作日志支持按类型、操作人、操作对象和时间查询。
- 待归属订单支持批量勾选、重新匹配、复制订单号和跳转配置。
- 页面操作必须归属到对应功能区：添加/编辑人员放在成员列表工具区，新建调整任务放在历史订单调整工具区，保存目标配置放在销售目标工具区；全局顶部不承载具体功能按钮。
- 业务单元配置拆分为“成员列表 / 店铺角色配置 / 操作日志”：成员列表只维护人员准入与业务职能；店铺角色配置以店铺为锚点维护销售负责人、客服负责人和业务单元归属。
- 店铺角色配置承载订单归属主链路：订单命中店铺后，同时识别销售负责人、客服负责人和业务单元；销售负责人用于业绩、销售目标和业务单元归属，客服负责人用于售后责任与客服绩效。
- 待归属订单的“去配置”跳转到店铺角色配置，而不是成员列表，避免把店铺销售/客服责任散落在人员编辑表单内。
- 新店铺录入后进入“待认领新店铺”辅助抽屉，不强行挂在左侧某个业务单元；运营负责人在任务池内认领到业务单元后再配置销售与客服角色。
- 店铺跨业务单元转移通过“转移BU”抽屉完成，必须选择目标业务单元、目标销售/客服、生效日期、转移原因和历史订单处理方式，默认仅影响新订单。
- 用户不知道店铺属于哪个业务单元时，通过“全局查找店铺”按钮打开辅助抽屉跨公司搜索；全局检索不能作为当前业务单元页面的同级页签，避免打断左侧树上下文。
- 店铺角色配置必须覆盖异常场景：新店铺未认领、客服未配置、负责人失效、业务单元停用、重复店铺、历史生效期重叠、跨 BU 转移待确认。
- 历史订单调整通过全屏抽屉完成范围圈定、明细核对、规则设置和危险确认。
- 历史订单调整 Step 1 支持“按订单号修改”和“按搜索条件修改”；按搜索条件修改包含下单时间、所属店铺、原运营、原业务单元、平台和最大影响量。
- 历史订单调整 Step 3 支持运营与业务单元两个维度分别选择“按当前规则匹配”或“指定值”，并展示覆盖口径和审计风险提示。
- 销售目标矩阵区分已锁定月份和可编辑月份；未来月份需点击“编辑”后调整，失焦不保存，必须点击“暂存”或按 Enter 才进入待提交状态。
- 销售目标暂存后显示“已暂存”标记，最终仍需点击页面主按钮“保存目标配置”并通过强弱校验。
- 左侧架构导航切换业务单元后，不改变当前模块；在销售目标模块中加载对应业务单元的年度目标设置、BU 行与汇总口径。
- 销售目标设置逻辑为“业务单元总盘子 -> 人员承接 -> 店铺可选拆解”：业务单元设置年度总目标，人员目标之和不能超过业务单元目标；如事业部要求拆到店铺，则店铺目标必须平齐所属人员目标。
- 销售目标矩阵按树表格展示，支持 BU 行展开/收起人员、人员行展开/收起店铺；展开状态不影响汇总列校验结果。
- 目标差额集中在全年汇总列展示：BU 行展示待分配或人员超额，人员行展示店铺拆解平齐或店铺差额，店铺叶子行只展示店铺目标，不再误用“拆解平齐”。
- 销售目标支持“表格精修 + 模板批量导入”两种路径；导入抽屉包含下载模板、上传文件、结果反馈三阶段。效率、准确性、易用性、反馈四项原则只作为设计验收标准，不作为正式页面可见内容。
- 年度目标导入模板按业务单元、人员、店铺三层填写，校验模板结构、单行数据格式、业务逻辑关系、拆解平衡和已结账月份保护。
- 年度目标上传后必须先做预检；如出现人员-店铺拆解不平、人员目标超过业务单元目标等强校验错误，导入动作必须阻断，只允许下载预检报告并重新上传修正文件。
- 年度目标上传后创建后台导入任务，用户通过任务中心追踪处理进度；不论成功、失败或部分成功，都从任务中心下载导入结果文件。
- 错误报告默认采用方案二：单 Sheet + 状态列，在原始数据右侧追加“是否导入成功”和“错误原因”两列，便于用户定位并修正。
- 上传规范截图的知识已转译为可操作流程、校验状态、结果文件和任务中心反馈；不展示蓝底原则容器、四原则卡片、产品思考提示等分析层内容。

## 4. 状态与权限

| 状态/权限 | 表达方式 |
|-----------|----------|
| 未勾选批量操作 | 按钮禁用，并显示启用条件提示 |
| 店铺未配置 | 成员表店铺列显示“待配置”标签 |
| 他人店铺强勾 | 店铺资源池行浅黄色高亮，保存时二次确认 |
| 高风险移除 | Modal 强警告，说明店铺会进入待归属订单池 |
| 历史覆盖订单 | Modal 强确认，说明永久记录审计日志 |
| 销售目标编辑权限 | 页面文案与操作入口按负责人/上级负责人语义表达，不提供角色切换器 |
| 空态 | 待归属订单表保留空态文案位置 |

## 5. 设计基线

- Figma 权威库：`Ant Design ERP UI Library`，fileKey `KaI3eGyylfiwrPlU3OR08C`，版本 `v0.2.1`。
- HTML 原型使用静态 `index.html` + `data.js`，不连接真实接口。
- 风格遵循中文 B 端 ERP：浅灰背景、白色内容面板、紧凑表格、Ant Design 蓝、4-8px 圆角。
- 正式页面内没有角色切换器、状态切换器、调试面板或演示控制区。
